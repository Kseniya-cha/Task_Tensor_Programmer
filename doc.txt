В решении используются библиотеки openpyxl для чтения и обработки файлов xlsx и os для получения всех файлов директории.

Критерий успешности сотрудников - сумма разностей между планируемыми человеко-днями и фактическими для всех проектов этого сотрудника. Успешности сотрудников, упомянутых
более чем в одном файле, складываются и делятся на количество проектов, в которых участвовал этот сотрудник. В результате идёт сортировка сотрудников по их успешности
из всех доступных для обработки файлов. ФИО сотрудников выводятся построчно.

Первый столбец обязательно содержит название проектов, иначе выводится сообщение о некорректности структуры страницы.

Отчёт о выполнении работы программы содержится в файле report.txt, создаваемом автоматически. Перед каждым выполнением программы файл очищается.

Класс DataFileXLS содержит методы для обработки одного файла. Функция success_by_one_file обрабатывает один файл. На вход принимается название файла с расширением xlsx.
Последующие две функции обрабатывают все xlsx-файлы из директории.

Реализована проверка корректности данных в ячейках с ФИО сотрудников (должны иметь формат "Фамилия И.О.", также данные в двух соседних ячейках, относящихся к одному
сотруднику, должны совпадать). Если перепутаны столбцы "план"/"факт", выводится сообщение с предупреждением, но предполагается, что перепутаны только ячейки с
названиями. Если по какому-либо критерию данные в ячейках с ФИО сотрудников не считаются корректными, выводится сообщение об этом и указание на название файла,
координаты обеих ячеек и данные, содержащиеся в них.

Для пар ячеек, содержащих человеко-часы, реализовано следующее: если значение одной из ячеек имеет не числовой тип, пара ячеек не обрабатывается и пропускается.

Если одна и та же строка дублируется 2 и более раз, все такие строки, кроме первой, зануляются. Дублируются в данном случае означают, что совпадает первая строка,
содержащая названия проектов, и строки с человеко-часами.

Если страница файла содержит некорректное число столбцов, об этом выводится сообщение и обработка не производится.

Если открыт один из обрабатываемых файлов, выводится сообщение о необходимости закрыть все файлы.

В v.3. добавлено всплывающее окно, спрашивающее пользователя, открыть ли файл с отчётом. При ответе "нет" окно закрывается, при ответе "да" - закрывается и открывается
файл 'report.txt'.
